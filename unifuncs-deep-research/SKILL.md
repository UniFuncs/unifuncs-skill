# UniFuncs 深度研究技能 (Deep Research)

## 定位说明

**深度研究 (Deep Research)** 是一个**专业级研究报告生成**工具：

- 📊 **万字报告**：自动生成结构化的专业研究报告
- 🔬 **多轮迭代**：深入挖掘信息，全面分析
- 📝 **多种格式**：支持报告、知乎文章、公众号文章等

---

## 触发条件

当用户请求符合以下模式时，触发此技能：

| 触发词 | 示例 | 说明 |
|--------|------|------|
| **dr** | "dr 大语言模型幻觉问题" | 简写触发 |
| **深度研究** | "深度研究 新能源汽车市场" | 完整触发词 |
| **deep research** | "deep research AI trends" | 英文触发 |
| **研究报告** | "帮我写一份关于 XXX 的研究报告" | 报告需求 |
| **万字报告** | "生成一份万字报告" | 长篇报告需求 |

**不触发此技能的情况**：
- "搜索 XXX" → 使用 `unifuncs-web`
- "ds XXX" / "深度搜索 XXX" → 使用 `unifuncs-search`
- 只需要简单了解某个主题 → 使用 `unifuncs-search`

## Step 1: 分析研究需求

### 参数选择策略

| 场景 | 输出类型 | 预期长度 | 其他参数 |
|------|----------|----------|----------|
| 专业研究报告 | `report` | 10000 | 默认 |
| 知乎专栏文章 | `zhihu-article` | 5000 | - |
| 公众号文章 | `wechat-article` | 3000-5000 | - |
| 小红书文章 | `xiaohongshu-article` | 1500-2000 | - |
| 快速摘要 | `summary` | 1000-2000 | - |

### 输出类型完整列表

| 类型 | 说明 | 建议长度 |
|------|------|----------|
| `report` | 万字深度报告（默认） | 10000+ |
| `summary` | 精炼摘要 | 1000-2000 |
| `zhihu-article` | 知乎专栏文章风格 | 3000-8000 |
| `zhihu-answer` | 知乎回答风格 | 1500-3000 |
| `wechat-article` | 微信公众号文章风格 | 2000-5000 |
| `xiaohongshu-article` | 小红书文章风格 | 1000-2000 |
| `toutiao-article` | 头条文章风格 | 2000-4000 |
| `weibo-article` | 微博长文风格 | 1000-2000 |

## Step 2: 执行深度研究

### 脚本调用

```bash
python3 ~/.claude/skills/unifuncs-research/scripts/research.py "[研究主题]" [参数]
```

> ⚠️ **注意**：深度研究耗时较长，通常需要 **2-10 分钟**，请耐心等待。

### 参数完整说明

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `topic` | - | 研究主题 (必填) | - |
| `--type` | `-t` | 输出类型（见上方列表） | report |
| `--length` | `-l` | 预期输出长度 | 10000 |
| `--depth` | `-d` | 研究深度 1-25 | 25 |
| `--reference` | `-r` | 引用格式: link/character/hidden | link |
| `--intro` | `-i` | 设定研究员的角色和口吻 | - |
| `--domain` | - | 限定搜索网站 (逗号分隔) | - |
| `--blacklist` | `-b` | 排除网站 (逗号分隔) | - |
| `--prompt` | `-p` | 自定义输出提示词 | - |
| `--timeout` | - | API 请求超时时间(秒)，最低 1800 | 1800 |

---

## Step 3: 展示研究报告

深度研究 API 会返回**完整的研究报告**，包含：

- 📋 **结构化内容**：按章节组织的专业报告
- 🔗 **引用来源**：信息的出处链接
- 📊 **深度分析**：多角度的专业分析

**直接展示 API 返回的内容即可**。

## 环境配置

### 必需的环境变量

```bash
export UNIFUNCS_API_KEY="sk-your-api-key"
```

获取地址：[UniFuncs 账户页面](https://unifuncs.com/account)

### 验证配置

```bash
# 检查环境变量是否已配置
echo $UNIFUNCS_API_KEY
```

如果输出为空，说明未配置，请设置环境变量。

---

## 错误处理

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| 未设置 UNIFUNCS_API_KEY | 环境变量未配置 | 设置 `export UNIFUNCS_API_KEY="sk-xxx"` |
| API 请求失败 (HTTP 401) | API Key 无效或过期 | 检查 API Key 是否正确 |
| API 请求失败 (HTTP 429) | 请求频率过高 | 稍后重试 |
| 超时 | 研究耗时过长 | 降低 depth 或 length 参数 |
