# UniFuncs 网页搜索与阅读技能

## 触发条件

当用户请求符合以下模式时，触发此技能：

| 触发词 | 示例 | 动作 |
|--------|------|------|
| "搜索 XXX"（不带"深度"） | "搜索 Python 教程" | 执行搜索流程 |
| "网页搜索 XXX" | "网页搜索 Claude API" | 执行搜索流程 |
| "读取网页 XXX" | "读取网页 https://example.com" | 直接阅读指定 URL |
| "查一下 XXX" | "查一下最新的 AI 新闻" | 执行搜索流程 |

**注意**：如果用户说"深度搜索"或"深度研究"，应使用 `unifuncs-search` 或 `unifuncs-research` 技能，而非本技能。

## Step 1: 执行搜索

### 脚本调用

```bash
python3 ~/.claude/skills/unifuncs-web/scripts/web_search.py "[关键词]" [参数]
```

### 参数选择策略

| 场景 | 推荐参数 |
|------|----------|
| 一般搜索 | `--count 10 --format markdown` |
| 需要最新信息 | `--freshness Day` 或 `--freshness Week` |
| 技术文档搜索 | `--count 15` （多获取一些结果以便筛选） |
| 中文内容优先 | `--area cn` |

### 参数完整说明

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `query` | - | 搜索关键词 (必填) | - |
| `--count` | `-c` | 每页结果数量 1-50 | 20 |
| `--page` | `-p` | 页码 | 1 |
| `--format` | `-f` | 输出格式: json/markdown/text | markdown |
| `--freshness` | - | 时效性: Day/Week/Month/Year | - |
| `--area` | `-a` | 搜索区域: global/cn | - |
| `--images` | `-i` | 包含图片搜索结果 | false |

## Step 2: 筛选结果并决定阅读

搜索返回结果后，**必须进行智能筛选**，选择最有价值的结果进行深入阅读。

### 筛选标准

**优先选择：**
1. **官方来源**：官方文档、官方博客、官方 GitHub
2. **权威站点**：知名技术博客、权威媒体、学术机构
3. **内容相关性**：标题和摘要与用户需求高度匹配
4. **时效性**：优先选择较新的内容（如有时效要求）

**避免选择：**
- 明显的广告或营销页面
- 内容农场或低质量聚合站
- 与用户需求不相关的结果

### 筛选数量建议

| 搜索结果数 | 建议阅读数 | 说明 |
|------------|------------|------|
| 1-5 条 | 1-2 条 | 选择最相关的 |
| 6-10 条 | 2-3 条 | 选择前几个最优质的 |
| 10+ 条 | 3-5 条 | 不要贪多，选择最有价值的 |

### 筛选输出格式

在阅读前，向用户简要说明筛选结果：

```
从搜索结果中筛选出以下 N 个高价值来源：
1. [标题] - [来源] - 选择原因
2. [标题] - [来源] - 选择原因
...

正在阅读这些页面以获取详细信息...
```

## Step 3: 阅读网页

### 脚本调用

```bash
python3 ~/.claude/skills/unifuncs-web/scripts/web_reader.py "[URL]" [参数]
```

### 参数选择策略

| 场景 | 推荐参数 |
|------|----------|
| 一般阅读 | `--format markdown` |
| 只需要正文 | `--lite` |
| 内容过长 | `--max-words 5000` |
| 提取特定主题 | `--topic "主题关键词"` |
| 需要页面链接 | `--links` |

### 参数完整说明

| 参数 | 简写 | 说明 | 默认值 |
|------|------|------|--------|
| `url` | - | 目标网页 URL (必填) | - |
| `--format` | `-f` | 输出格式: markdown/text | markdown |
| `--lite` | `-l` | 可读性过滤，只保留正文 | false |
| `--no-images` | - | 不包含图片 | false |
| `--max-words` | `-m` | 最大字符数限制 | 不限制 |
| `--timeout` | `-t` | 读取超时时间(毫秒) | 120000 |
| `--topic` | - | 提取特定主题相关内容 | - |
| `--links` | - | 附加页面链接列表 | false |

## Step 4: 整理并反馈

阅读完成后，**必须整理信息**并以结构化方式反馈给用户。

### 反馈格式模板

```markdown
## 搜索结果摘要

**搜索关键词**：[用户的搜索词]
**信息来源**：共阅读 N 个网页

---

### 核心发现

[根据阅读内容，提炼出最重要的信息点]

1. **要点一**：...
2. **要点二**：...
3. **要点三**：...

---

### 详细内容

#### 来源 1: [标题]
- **链接**：[URL]
- **摘要**：[关键内容摘要]

#### 来源 2: [标题]
- **链接**：[URL]
- **摘要**：[关键内容摘要]

---

### 补充说明

[如有需要，添加额外的说明或建议]
```

## 完整示例

### 示例 1：搜索请求


## 环境配置

### 必需的环境变量

```bash
export UNIFUNCS_API_KEY="sk-your-api-key"
```

获取地址：[UniFuncs 账户页面](https://unifuncs.com/account)

### 验证配置

```bash
# 检查环境变量是否已配置
echo $UNIFUNCS_API_KEY
```

如果输出为空，说明未配置，请设置环境变量。

---

## 错误处理

| 错误信息 | 原因 | 解决方案 |
|----------|------|----------|
| 未设置 UNIFUNCS_API_KEY | 环境变量未配置 | 设置 `export UNIFUNCS_API_KEY="sk-xxx"` |
| API 请求失败 (HTTP 401) | API Key 无效或过期 | 检查 API Key 是否正确 |
| API 请求失败 (HTTP 429) | 请求频率过高 | 稍后重试 |
| 网络错误 | 网络连接问题 | 检查网络连接 |
